# NEXT 16 + REACT 19 — read before writing ANY page/route/component

This project runs **Next.js 16.2.7 + React 19.2.4 + Tailwind v4**. These differ from older training data. The scaffold's own `web/AGENTS.md` says: read `web/node_modules/next/dist/docs/` before writing code. The gotchas that will actually bite this build:

## 1. `params` and `searchParams` are async (Promises) — `await` them
```tsx
// app/blog/[slug]/page.tsx
export default async function Page({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;   // ← MUST await
}
// search params:
export default async function Page({ searchParams }: { searchParams: Promise<Record<string,string|string[]|undefined>> }) {
  const sp = await searchParams;
}
```
Prefer the **global helper types** (no import needed, generated by `next dev`/`build`):
```tsx
export default async function Page(props: PageProps<'/blog/[slug]'>) {
  const { slug } = await props.params;
}
```
`generateMetadata` gets the same async params: `export async function generateMetadata({ params }: { params: Promise<{slug:string}> })`.
`generateStaticParams` is unchanged: `return posts.map(p => ({ slug: p.slug }))`.

## 2. Route handlers (API routes) — `app/api/**/route.ts`
```ts
import type { NextRequest } from 'next/server';
export async function GET(req: NextRequest) { return Response.json({ ok: true }); }
// dynamic segment → use the global RouteContext helper, await ctx.params:
export async function GET(_req: NextRequest, ctx: RouteContext<'/api/calendly/slots/[id]'>) {
  const { id } = await ctx.params;
}
```
- Route handlers are **NOT cached by default** (good — our Calendly routes are dynamic). Don't add `force-static` to them.
- `cookies()`, `headers()`, `draftMode()` are **async** — `await headers()`.

## 3. Components
- Server Components by default. Add `"use client"` at the top of any file using hooks, framer-motion, event handlers, `useInView`, etc.
- React 19: `useFormState` → `useActionState`; ref-as-prop allowed (no `forwardRef` needed in many cases). We mostly use client components with framer-motion — keep `"use client"`.
- `<Link>` from `next/link` unchanged. Use it for internal nav (never `<a href>` for internal routes).

## 4. Tailwind v4
- No `tailwind.config.js`. Tokens live in `globals.css` via `:root { --token: … }` + `@theme inline { … }` mapping. Use `var(--token)` in `style={{}}` for brand colors per design-system.md component rules; Tailwind utilities for layout/spacing.
- `@import "tailwindcss";` is already at the top of globals.css.

## 5. Metadata / SEO
- `export const metadata: Metadata = {…}` in layout/page (static) OR `export async function generateMetadata(...)` (dynamic, async params).
- `metadataBase` set in root layout. `sitemap.ts`, `robots.ts`, `opengraph-image.tsx` are special files in `app/` (static by default).

## 6. Images & video
- `next/image` `<Image>` unchanged for static imagery. The hero uses a raw `<video>` (see Hero.tsx) — full-bleed movie header, NOT next/image.

When unsure, read the matching file in `web/node_modules/next/dist/docs/01-app/` — it's the version-accurate source.
